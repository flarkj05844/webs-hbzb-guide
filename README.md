# webs-hbzb-guide

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态网页形式存在，不依赖后端服务，可直接在浏览器中打开或部署至任意静态托管平台。

仓库内容聚焦于各类信息展示、工具页面或文档型 HTML 文件，不针对任何特定域名或具体网站。每个页面均为独立单元，互不依赖。

## 目录说明

```
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 公共资源文件（CSS、JS、图片等）
├── index.html      # 仓库页面索引（可选）
└── README.md       # 本说明文件
```

- `pages/`：每个 HTML 文件独立存放，文件名体现页面内容。
- `assets/`：多个页面共用的样式、脚本或图片资源，按需组织子目录。

## 页面归档说明

- 所有页面均为静态 HTML，兼容主流浏览器。
- 页面之间无跨文件依赖，可单独使用。
- 每个页面内嵌必要的样式与逻辑，尽量减少对外部资源的依赖。
- 归档内容不定期更新，新增页面或修正已有页面。

## 维护说明

- 欢迎提交 Issue 反馈页面问题或提出改进建议。
- 如需新增页面，请确保为独立 HTML 文件，并放置于 `pages/` 目录。
- 提交 Pull Request 时请简要说明页面用途或修改内容。
- 请勿在页面中包含恶意代码、跟踪脚本或违反法律法规的内容。

## 使用方式

1. 克隆或下载本仓库。
2. 直接双击 `pages/` 下的 HTML 文件在浏览器中打开。
3. 或将整个仓库部署至任何静态托管服务（如 GitHub Pages、Netlify 等）。
